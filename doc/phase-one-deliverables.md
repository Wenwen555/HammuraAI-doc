# 第一阶段交付

## 阶段目标

第一阶段聚焦在 HammuraAI 的基础能力打底，不追求生产级完整闭环，而是完成一个可以被展示、被验证、被继续开发的法律 AI 后端框架。

## 交付范围

### 1. 数据库搭建

- 建立 PostgreSQL 连接与异步访问层
- 明确核心 ORM 模型
- 通过 Alembic 承担迁移，不在运行时自动建表
- 明确 PostgreSQL + pgvector 一体化 RAG 底座路线
- 设计 `documents` / `document_chunks` 目标表结构与向量索引迁移方案

### 2. 检索工具

- 接入法规索引与案例索引
- 通过 Embedding 微服务完成语义向量计算
- 提供基础检索能力与索引状态检查
- 验证 metadata filter + pgvector Top-K 的混合检索路径
- 保留关键词 / 全文检索作为法律术语和法条编号的补充召回

### 3. 重排与引用

- 对召回结果做轻量重排
- 输出 `citations` 结构
- 输出 `rag_block` 供前端和下游生成使用
- 将向量召回、关键词召回和业务规则偏置融合为统一候选排序

### 4. LLM 调用模块

- 建立统一的模型选择与调用入口
- 支持同步文本生成和流式输出
- 支持结构化场景摘要生成
- 消费检索上下文、引用结构和拒答约束，生成带来源的法律回答

### 5. 用户与会话

- 完成登录、会话 Cookie、用户资料聚合
- 完成多用户会话实体与消息持久化
- 完成 SSE 流式会话运行时

### 6. Prompt 模板管理

- 建立集中式 prompts 文件
- 明确法律咨询、追问、总结、上下文注入模板
- 预留分层治理空间

## 当前边界

第一阶段暂不承诺：

- 案件级强权限隔离
- 全量私有知识库治理
- 生产级审计闭环
- 多 Agent 自动化编排
- 商业法律数据库接入

## 第一阶段完成标志

1. 服务可启动，核心依赖可连接。
2. 一条典型法律问答链路可跑通。
3. 回答可附带最小引用来源。
4. 技术文档和代码结构可以继续支撑第二阶段扩展。
