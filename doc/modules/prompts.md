# Prompt 模板管理

## 模块目标

Prompt 模板管理用于保证 HammuraAI 在法律场景下的输出风格、追问策略、引用规则和专业边界保持一致。

它不只是“写几段提示词”，而是整个法律智能交互的规则层。

## 当前实现

核心文件：

- `backend_app/app/services/prompts.py`
- `backend_app/app/services/context_engine.py`

## 当前能力

### 模板集中管理

目前 prompts 已经集中在单个服务文件中，覆盖：

- 法律咨询对话
- 前端摘要
- 后续追问问题
- 文件上下文注入
- RAG 上下文注入
- 案件结构化引导

### 隐藏上下文注入

当前系统明确区分两类隐藏上下文：

- 文件摘要上下文
- 外部检索上下文

这对于法律场景非常关键，因为系统既要利用材料，又不能把原文长段复述出去。

### 专业法律引导

Prompt 已经承担以下职责：

- 引导用户补齐案件事实
- 约束回答不编造
- 输出自然中文摘要
- 保持法律分析的结构化倾向

## 第一阶段交付重点

1. 建立统一 Prompt 入口。
2. 支撑法律咨询、追问、摘要、RAG 注入四类关键场景。
3. 为后续版本治理保留扩展空间。

## 当前不足

- 文件体量偏大，后续应拆分治理
- 模板版本与回归验证尚未形成机制
- 不同任务类型的边界仍有继续收敛空间

## 后续演进方向

- 按任务域拆分 prompts 文件
- 建立模板清单和版本记录
- 把法律请求引导模板显式制度化
