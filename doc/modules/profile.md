# 用户档案管理

## 模块目标

用户档案管理负责支撑 HammuraAI 的“谁在用、处于什么套餐、剩余多少额度、能看到哪些历史信息”这条业务面。

## 当前实现

核心文件：

- `backend_app/app/api/v1/endpoints/auth.py`
- `backend_app/app/api/v1/endpoints/profile.py`
- `backend_app/app/models/database.py`

## 当前能力

### 用户认证

- 支持注册、登录、Session Cookie
- 通过 Redis 维护会话状态
- 自动补基础套餐记录

### 个人中心

当前已支持：

- 资料聚合
- 用户名与头像修改
- 密码修改
- 用量统计
- 积分流水查看

### 套餐与钱包

已有数据层覆盖：

- `Plan`
- `Subscription`
- `Usage`
- `Wallet`
- `CreditLedger`

这让 HammuraAI 后续可以把法律问答能力与用户权益绑定起来。

## 第一阶段交付重点

1. 用户身份可以稳定识别。
2. 会话能关联到具体用户。
3. 档案和权益信息有统一读取入口。

## 当前不足

- 还没有更细粒度的法律角色体系
- 案件级授权与资料级授权尚未深入
- 前端侧的档案体验还不是本阶段重点

## 后续演进方向

- 引入角色和组织级结构
- 接入案件、客户、团队三层关系
- 建立法律服务场景下的可审计用户画像
